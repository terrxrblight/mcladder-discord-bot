# mcladder-discord-bot

Discord-бот для рейтингового Minecraft-сервера **mcladder.com**.

**Что уже умеет (этап 1):**
- 🏆 **Живой лидерборд** — одно закреплённое сообщение в канале, авто-обновление топ-10. Данные тянутся из публичного API `mcladder.com/api`, поэтому **синхронно с игрой и сайтом** (та же сортировка, те же тай-брейкеры). Перерисовывается только когда данные реально изменились (поллинг `/api/version`).
- 📊 `/stats` — карточка игрока (место, ELO/пик, W-L, винрейт, тир, статус, бан, бейджи). По нику (`player:<ник>`), по `@user` (если он привязал MC на сайте) или своя без аргументов (если привязан).
- 🟢 **Статус бота** — «Watching N online · M in match» (онлайн из `player_presence`).
- 🎖️ **Авто-роли** — роль по тиру (Wood..Netherite) + **Verified** тем, кто привязал MC на сайте. Привязка читается из `user_identities` (отдельный /link не нужен). Недостающие роли бот создаёт сам.
- 🔨 **Мод-лог банов** — новые баны/разбаны из таблицы `bans` → админский `#mod-logs`.
- ⚔️ **Лента матчей** — для каждой завершённой рейтинговой катки в `#match-results` бот рисует **картинку-карточку**: фон — заблюренная карта матча, поверх — головы игроков, ники и изменение ELO. Плюс кликабельные ники и ссылка на матч в тексте эмбеда. Без бэкфилла: постит только новые. Включается через `MATCH_RESULTS_CHANNEL_ID` (пусто → выключено). Картинка рисуется через `@napi-rs/canvas` (после обновления нужен `npm install`); если пакет недоступен — карточка автоматически откатывается на текстовый вид.
- 🎖️ **Эмодзи рангов** — бот подтягивает кастом-эмодзи гильдии (`:wood:`…`:netherite:`, `:unrated:`) и показывает их в лидерборде/`/stats`/карточках.
- 🔧 `/refreshboard` — принудительное обновление лидерборда (только для админов).

**Чат / join-leave** делает отдельный плагин **RankedDiscord** (в `../bridge`).

---

## Требования
- Node.js ≥ 18.17 (на VPS уже стоит — тот же, что у сайта)
- Доступа к БД **не требуется** — бот работает через публичный API.

## Установка

```bash
cd bot
npm install
cp .env.example .env   # затем заполни .env (см. ниже)
```

## Настройка Discord (один раз)

1. **Создай приложение и бота:**
   - https://discord.com/developers/applications → **New Application**.
   - Вкладка **Bot** → **Reset Token** → скопируй токен в `DISCORD_TOKEN`.
   - Вкладка **General Information** → скопируй **Application ID** в `DISCORD_CLIENT_ID`.
   - Привилегированные интенты (Message Content и пр.) **не нужны** — оставь выключенными.

2. **Пригласи бота на сервер:**
   - Вкладка **OAuth2 → URL Generator** → scopes: `bot` + `applications.commands`.
   - Bot Permissions: **View Channel**, **Send Messages**, **Embed Links**, **Manage Messages** (чтобы закреплять лидерборд).
   - Открой получившийся URL → добавь бота на свой сервер.

3. **Получи ID сервера и канала:**
   - В Discord включи **Settings → Advanced → Developer Mode**.
   - ПКМ по серверу → **Copy Server ID** → в `GUILD_ID`.
   - Создай канал (напр. `#leaderboard`), ПКМ → **Copy Channel ID** → в `LEADERBOARD_CHANNEL_ID`.
   - Убедись, что у бота есть доступ к этому каналу.

## Запуск

```bash
# 1) зарегистрировать слэш-команды (после каждого изменения набора команд)
npm run deploy-commands

# 2) старт
npm start
```

Бот залогинится, опубликует лидерборд в канале и будет держать его свежим.

### Проверка логики без Discord
Печатает, что попадёт в эмбеды (тянет живой API, токен не нужен):
```bash
npm run dry-run
```

## Боевой запуск на VPS (PM2)

```bash
npm install --omit=dev
npm run deploy-commands
pm2 start ecosystem.config.js
pm2 save        # чтобы пережить ребут (если pm2 startup уже настроен для сайта)
pm2 logs mcladder-bot
```

## Переменные окружения
См. `.env.example`. Кратко:

| Переменная | Назначение |
|---|---|
| `DISCORD_TOKEN` | токен бота |
| `DISCORD_CLIENT_ID` | Application ID |
| `GUILD_ID` | ID сервера |
| `LEADERBOARD_CHANNEL_ID` | канал лидерборда |
| `MATCH_RESULTS_CHANNEL_ID` | канал ленты матчей (пусто → выключено) |
| `LEADERBOARD_LIMIT` | сколько мест (по умолчанию 10) |
| `POLL_INTERVAL_MS` | период опроса свежести (по умолчанию 15000) |
| `API_BASE` / `SITE_BASE` | адреса API/сайта (по умолчанию прод) |

## БД-фичи (этап 3): авто-роли, Verified, мод-лог, онлайн
Эти фичи читают Postgres под `ranked_read` (бот на том же VPS → `127.0.0.1`).

1. Заполни в `.env` секцию `DB_*` (креды `ranked_read` возьми из `RankedChat/config.yml`).
2. Для авто-ролей: `ROLE_SYNC=true`, в Dev Portal включи **Server Members Intent**,
   у бота должно быть право **Manage Roles**, а роль бота — **выше** управляемых
   ролей (Verified, Wood..Netherite). Недостающие роли бот создаёт сам с цветом тира.
3. Для мод-лога: создай админский канал и впиши его ID в `MOD_LOG_CHANNEL_ID`.
4. `npm install` (добавился `pg`) → `pm2 restart mcladder-bot`.

Без `DB_*` все БД-фичи просто не запускаются — лидерборд/лента/`/stats <ник>` работают как раньше.

├── src/
